# Infrastructure Services

Infrastructures Services are not part of the main platform, but are used by it.

# Edge

Order of requests:

  1. The internet makes requests through a cloud provided load balancer.
  2. That load balancer passes all requests through to the Ingress on Kubernetes.
  3. The Ingress terminates HTTPs and routes requests a) To a specific se
  4. Based on the path of the request Nginx will route traffic to the appropriate service.

# ingress

Ingress (opens new window) is a Kubernetes concept that exposes HTTP and HTTPS routes from outside the cluster to services within the cluster.

# cert-manager

cert-manager (opens new window) provides automate certificate management in Kubernetes.

This is used in conjunction with the ingress.

# platform-nginx

Nginx (opens new window) is used as a reverse proxy layer to perform path based routing to difference services for sites hosted on wbstack.

Requests to platform services such as the main UI or API do not go via this layer.

# sql

# redis

Redis is used by a variety of deployed services.
